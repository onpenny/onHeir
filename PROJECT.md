# OnHeritage - 遗产管理和安排平台

## 项目概述
一个全生命周期的遗产管理平台，解决资产信息孤岛化、继承主体不明确、生前资产遗忘漏记、继承流程非标繁琐等核心痛点。

---

## 一、当前遗产管理核心痛点

1. **资产信息孤岛化**：银行、保险、券商、基金、不动产、虚拟资产等分属不同机构，无统一归集入口，机构无主动告知继承人的法定义务与通道，大量沉睡资产无法被继承人查知。

2. **继承主体不明确**：无标准化家族谱系存储机制，旁系继承人、非婚生子女、遗赠对象等信息易断层，出现继承权确权难、主体认定纠纷。

3. **生前资产遗忘漏记**：多类投资、另类资产（收藏品、股权、知识产权）零散分布，用户生前自身都难完整盘点，身后更无迹可寻。

4. **继承流程非标繁琐**：不同地区 / 法域继承法律、公证、过户流程差异大，无一站式指引，继承人时间、经济成本极高。

5. **遗产捐献无闭环**：捐赠意愿登记、机构对接、凭证留存、进度追溯缺乏统一平台，意愿落地难、透明度低。

6. **跨境继承壁垒高**：海外银行、保险、不动产等资产，受法律、语言、机构壁垒限制，继承人几乎无自主查询路径，资产流失率高。

7. **隐私与存储矛盾**：传统纸质 / 零散电子存储易泄露、破损，兼具安全性与可继承性的数字化解决方案稀缺。

## 二、核心基础功能

### 2.1 全品类资产管理
- 对接金融机构 API，实现银行、保险、理财等资产自动同步入账
- 支持手动录入非标资产：股权、收藏品、知识产权、虚拟货币、海外资产
- 生前定期资产盘点提醒，降低遗忘、漏记概率

### 2.2 智能家族谱系追踪
- 可视化族谱搭建，兼容血缘、收养、婚姻等多元关系标注
- 预设法定的 / 指定继承人层级，关联资产分配规则
- 分级权限管控，仅授权主体可查阅敏感信息

### 2.3 自动触发与继承通知
- 合规对接医疗、户籍、社保数据，核验生命状态并触发后续流程
- 按预设规则自动向继承人推送资产清单、继承指引
- 支持条件触发通知（子女成年、完成学业等）、延迟通知

### 2.4 遗嘱与分配规则管理
- 提供多地区 / 法域合规遗嘱模板
- 可视化资产分配比例设置，支持动态条件分配
- 区块链存证，强化电子数据法律效力

### 2.5 遗产捐献全流程管控
- 财产、文物、遗体 / 器官等捐赠意愿登记
- 对接正规基金会、医疗机构、文博机构，直连执行端
- 捐赠凭证、资金流向、项目进度实名追溯 / 匿名公示

### 2.6 一站式继承工具
- 法域适配继承材料清单、流程指引、文档模板
- 在线公证、律师咨询预约入口
- 跨境继承法律条款检索与基础导航

### 2.7 隐私安全体系
- 端到端加密、操作留痕、多重身份验证（生物识别、委托联系人核验）
- 多节点备份，防止数据丢失与泄露

## 三、核心商业盈利点

### 3.1 订阅制收费（稳定基础现金流）
- **个人分层订阅**：免费版、基础年付版、进阶年付版、高端尊享版
- **家族套装订阅**：捆绑多成员账号，统一族谱与资产池管理

### 3.2 高毛利增值服务收费
- **法律与公证服务**：遗嘱见证、继承咨询、纠纷代理、跨境法域法律意见书
- **遗产清算与代执行**：资产核验、变现、过户代办
- **非标资产评估**：收藏品、艺术品、股权、知识产权评估对接
- **税务筹划服务**：遗产税、赠与税优化方案设计
- **沉睡资产寻回**：历史资产专业查询、确权、追回代办（成功追回后按比例分成）

### 3.3 B 端合作授权与分成（规模化收入）
- **金融机构合作**：向银行、保险、信托、券商输出白标系统模块
- **政企与机构项目制**：为民政部门、公证处、公益基金会定制的谱系 / 捐赠 / 继承管理系统
- **跨境服务对接**：联合海外律所、跨境财富机构，收取居间费 + 成功分成

### 3.4 衍生变现触点
- **数字遗产与文创**：社交账号、数字作品、影像托管与展示；族谱画册、家族传承档案等文创实物销售
- **行业数据服务（合规匿名化）**：输出遗产分布、继承风险、捐赠意向等行业报告
- **精准产品导流**：生前规划场景合规推荐终身寿险、家族信托等产品

---

## 四、差异化竞争亮点

1. **谱系 + 资产双绑定**：区别于单一资产管理 / 族谱工具，实现关系与分配规则深度关联
2. **主动触发机制**：不同于传统被动查询平台，通过合规数据核验实现主动通知
3. **全法域适配**：覆盖境内多地区 + 主流海外国家规则，抢占跨境继承蓝海
4. **捐赠全闭环**：从意愿登记到执行追溯一体化
5. **生前死后全周期**：覆盖规划、预案、执行、传承，非单一环节工具

---

## 五、合规与风险把控

- 严格遵循各地数据安全、个人信息保护法规（如 PIPL、GDPR）
- 明确电子遗嘱、区块链存证法律效力边界
- 仅与持牌法律、金融、评估机构合作
- 建立完善的授权、注销、数据删除机制
- 流程全留痕，降低平台在继承纠纷中的法律责任

---

## 六、冷启动获客思路

1. 联合律师事务所、家族办公室、保险经纪人，精准触达高意向、高价值用户
2. 推出「免费沉睡资产排查」活动
3. 针对中老年、高净值群体做继承风险、资产流失案例科普
4. 与殡葬、寿险场景捆绑，作为增值服务打包切入
5. 优先突破港澳台及跨境用户

## 技术栈

### 前端
- **框架**：Next.js (App Router)
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **组件库**：Shadcn/ui（基于 Radix UI）

### 后端 & API
- **API**：Next.js API Routes
- **数据库 ORM**：Prisma
- **认证**：NextAuth.js v5

### 数据库
- **开发/早期生产**：SQLite
  - ✅ 无需安装，文件存储，快速启动
  - ✅ 适合快速迭代和开发
  - ✅ 后来可以轻松迁移到 PostgreSQL
- **生产环境（未来）**：PostgreSQL
  - 更好的并发性能
  - 更适合高负载场景

### 状态管理
- **全局状态**：Zustand
- **表单处理**：React Hook Form + Zod
